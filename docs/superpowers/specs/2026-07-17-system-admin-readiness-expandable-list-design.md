# System Admin 配置完成度可展开列表设计

## 目标

将“配置完成度”页面从纵向堆叠的大卡片改为紧凑的店铺列表，让运营人员先快速扫描全部店铺，再按需查看具体未通过项。

## 信息架构

- 列表以店铺为主行，每家店铺只占一行。
- 主行展示：展开控制、店铺 ID、租户 ID、状态、未通过项数量。
- 默认全部收起；点击展开控制后，在该店铺主行下方显示检查项明细。
- 明细以列表/表格行展示检查项、原因、影响和下一步，只展示状态不是 `pass` 的检查项。
- 全部检查通过时，展开区域显示“所有上线检查均已通过”。
- 保留现有服务端分页和“共 N 家店铺”口径，不新增接口、筛选或批量操作。

## 交互与可访问性

- 展开控制使用真实按钮，并通过 `aria-expanded` 表达当前状态。
- 按店铺 ID 维护展开状态；翻页返回新数据时，不错误复用其他店铺的展开内容。
- 店铺 ID 和租户 ID 保留完整值与可换行能力，长 ID 不造成横向溢出。
- 桌面端使用紧凑列式布局；移动端主行和明细按字段标签重排为纵向信息块，不依赖横向滚动。

## 状态与错误边界

- 加载、空、无权限、局部失败和请求失败继续复用现有 `RequestStateView`。
- 状态值和完成度数据继续完全来自 System Admin API，不增加前端推断或演示数据。
- 本次只改变展示结构，不改变 `blocked`、`warning`、`ready` 的服务端语义。

## 验收

- 页面初始只显示每家店铺的紧凑主行，不显示检查明细。
- 展开单家店铺后，只出现该店铺的未通过检查项；再次点击后收起。
- 多家店铺可独立展开和收起。
- 只保留一个分页控件，分页请求参数与当前实现一致。
- 桌面 `1440x900` 和移动 `390x844` 均无横向溢出。
- System Admin 组件测试、Admin UI 回归检查和生产构建通过。
