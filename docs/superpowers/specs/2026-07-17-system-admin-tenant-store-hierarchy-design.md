# System Admin 租户与店铺统一层级列表设计

## 目标

将 System Admin“租户与店铺”页面现有的租户表和店铺表合并为一个层级列表，让租户与所属店铺的关系在同一浏览路径中清晰可见，同时保留现有详情抽屉能力。

## 页面结构

- 页面保留“租户与店铺”标题，仅展示一个统一列表。
- 顶部汇总同时显示当前数据集中的租户总数和店铺总数，例如“共 8 个租户、9 家店铺”。
- 列统一为“名称 / ID”“类型”“平台”“状态”“创建时间”。
- 租户作为主行，展示租户名称、客户 ID、状态、创建时间和所属店铺数量。
- 店铺作为缩进子行，展示店铺 ID、平台、状态和创建时间；类型列明确显示“店铺”。
- 租户默认展开。点击租户行的展开控件可收起或展开其店铺，控件必须提供可访问名称及 `aria-expanded` 状态。
- 点击租户或店铺的数据区域继续打开对应的详情抽屉；展开控件不得误触发详情抽屉。
- 没有店铺的租户仍显示主行，并以“暂无店铺”子状态说明。

## 数据与分页

- 前端继续使用现有租户和店铺接口，不新增或修改后端 API。
- 页面加载当前租户页后，根据租户 ID 将已取得的店铺记录归组到各租户下。
- 统一列表只保留租户分页控件；租户主行及其全部店铺必须位于同一页，不允许跨页拆分。
- 为避免现有独立店铺分页导致归组不完整，加载租户页时按当前接口允许的页大小取得覆盖全部店铺的结果。若服务端店铺总数超过单次上限，则前端继续请求后续店铺页，直至取得完整店铺集合。
- 切换租户页时不重复展示或遗漏租户；店铺总数使用服务端分页元数据的 `total`。
- 任一接口失败时沿用当前部分失败状态：保留成功的数据并明确提示缺失的数据源，不把空数组误报成真实无数据。

## 视觉与响应式

- 延续 System Admin 的 IBM / Carbon 式高密度表格、细分隔线和低阴影，不引入卡片化视觉。
- 店铺子行通过缩进、层级引导线或轻微背景差异表达归属，不依赖颜色作为唯一层级信号。
- 桌面端保持完整列；移动端允许次要字段换行或隐藏，但不得产生水平溢出。
- 长租户 ID 和店铺 ID 必须可换行或截断并提供完整值，不能撑破表格。

## 测试与验收

- 组件测试先验证租户与所属店铺渲染在同一个列表中，且旧的两张独立表和店铺分页不再出现。
- 覆盖默认展开、收起/展开、无店铺租户、租户与店铺详情抽屉、长 ID 和统一总数文案。
- 数据加载测试覆盖店铺多页聚合以及租户/店铺接口部分失败。
- 运行 System Admin 组件测试、Admin Web 边界测试、TypeScript 构建。
- 在 `1440x900` 与 `390x844` 检查真实页面，确认无水平溢出、System Admin 仍只调用自身鉴权接口。

## 非目标

- 不修改租户、店铺的数据模型或 API 契约。
- 不在本次增加搜索、筛选、排序、批量操作或租户/店铺编辑能力。
- 不改变“配置完成度”页面或 Customer Admin。
