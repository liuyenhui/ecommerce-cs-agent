# System Admin LLM 配置交互优化设计

## 背景与目标

System Admin 的“LLM 配置”当前把可用模型编辑器和 LangGraph 节点绑定纵向堆叠在同一页，编辑表单占据列表上方空间，绑定保存按钮也无法准确表达未修改、保存中和保存失败状态。本次优化保持现有 System Admin 身份、权限和安全边界，只改善信息架构、编辑反馈与删除治理。

## 页面信息架构

页面保留“LLM 配置”一级标题，正文使用两个二级 Tab：

- `LLM`：管理可用模型、API Key 替换、连接测试、启停和删除。
- `LangGraph`：管理真实 LangGraph 节点与 LLM 的绑定。

Tab 状态写入 URL 查询参数 `tab=llms|langgraph`。缺失或非法值回退到 `llms`；刷新、前进后退和复制链接均保留当前 Tab。Tab 使用可键盘操作的 `tablist/tab/tabpanel` 语义，移动端允许横向滚动但不得造成页面横向溢出。

## LLM 列表与编辑弹窗

“可用 LLM”标题同行展示括号小字“（API Key 仅通过 HTTPS 提交一次，之后只显示掩码）”，右侧保留“添加 LLM”。添加和编辑都使用焦点受控的模态弹窗，支持 Escape、遮罩关闭、取消按钮和关闭后焦点返回触发按钮；保存中禁止关闭和重复提交。

编辑弹窗包含名称、厂商、Base URL、模型 ID 和 API Key。API Key 字段显示服务端返回的掩码值，例如 `••••0c59`，绝不返回或显示完整明文。掩码值仅用于展示，不作为更新 payload；用户输入新值后才提交 Key 替换，留空或未触碰均保留原 Key。成功后关闭弹窗并刷新列表；失败时保留用户输入并显示字段级或表单级错误。

## 删除治理

LLM 行操作增加“删除”。删除使用二次确认弹窗：

- 没有当前 LangGraph 绑定、配置版本或其他受保护引用时，允许永久删除 LLM 配置，并写 System Admin 审计。
- 存在引用时，服务端返回 `409` 和不含敏感内容的引用分类/数量；前端阻止删除并提示先解除绑定或停用。
- 不级联删除配置版本、调用指标、发布记录或审计记录。

后端新增 `DELETE /v1/system-admin/llms/{llm_id}`。权限沿用 LLM 写权限；不存在返回 `404`，并发状态变化按服务端最终检查结果处理。OpenAPI、服务、存储库和契约测试必须同步更新。

## LangGraph 绑定交互

“LangGraph 节点使用的 LLM”标题同行展示括号小字“（全系统配置，保存后立即作用于新请求）”。页面加载后保存一份规范化的初始绑定快照；只有当前选择与快照不同且没有正在保存时，“保存全部绑定”才可用。

保存期间按钮显示 loading 文案和忙碌状态，并禁止重复提交或继续修改。成功后用服务端响应更新快照并恢复 disabled；失败则保留本地修改、恢复按钮可用并显示错误。切换 Tab 不丢失未保存修改；离开页面或刷新前出现未保存修改提示。

## 安全与权限

- Customer Admin 与 System Admin 的站点、Cookie、路由守卫和 API 域保持隔离。
- 完整 API Key 不进入 GET 响应、React state、DOM、日志、审计、错误信息或测试快照。
- 掩码 Key 只能作为展示文本；更新请求只携带用户新输入的 Key。
- 无写权限角色不显示添加、编辑、启停、删除或保存绑定操作；连接测试继续遵循现有独立权限。
- 删除引用警告只返回类型和数量，不返回配置正文、Prompt、消息或 Secret。

## 测试与验收

- 组件测试覆盖 Tab URL 同步、弹窗焦点/关闭、掩码 Key 不回传、dirty/loading/error/success 状态、删除无引用成功及有引用 `409` 警告。
- API 与契约测试覆盖删除权限、`404`、引用冲突、无级联删除和审计。
- 桌面 `1440x900` 与移动 `390x844` 验证无横向溢出、Tab 可访问、弹窗可操作、保存状态清晰。
- 验证 System Admin 只调用 `/v1/system-admin/*`，不出现 Customer Admin 入口或 `/v1/admin/auth/me` 请求。
- 提交前运行受影响测试、全量测试、contract、diff check 和 Secret 扫描。

## 非目标

- 不展示或恢复完整 API Key。
- 不改变 LLM node-binding 运行语义、模型调用安全 validator 或发布流程。
- 不级联清理历史版本和用量数据。
- 不部署或合并；实现完成后通过独立分支和 PR 交付。
