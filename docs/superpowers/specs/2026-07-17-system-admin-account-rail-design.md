# System Admin 账号摘要侧栏布局设计

## 目标

把 System Admin 的“当前账号”摘要从正文右侧移到左侧导航底部，释放正文横向空间，同时保持桌面折叠导航和移动导航抽屉可用。

## 设计

- `AdminFrame` 提供可选的侧栏底部内容插槽；System Admin 将现有 `SystemUserSummary` 放入该插槽，不复制账号数据或请求。
- 展开侧栏时，账号摘要显示在“收起”按钮下方；折叠侧栏时隐藏摘要，避免 64px 宽度内产生不可读内容。
- 移动端导航抽屉保持展开宽度，账号摘要随导航滚动并显示在底部。
- `SystemWorkspace` 仅渲染页面正文，删除原来的右侧账号列，正文占满可用宽度。
- 不修改 System Admin 登录、session、角色或 API 调用边界。

## 验收

- 登录后账号摘要位于 `.rail` 内，且主内容 `.systemWorkspace` 内不再存在账号摘要。
- 桌面折叠导航时账号摘要隐藏；重新展开后恢复。
- 移动导航中账号摘要可见，页面无横向溢出。
- 现有 System Admin 单元测试、UI 回归检查和生产构建通过。
