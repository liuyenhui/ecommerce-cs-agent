# System Admin 未满足上线条件文案设计

## 目标

消除系统总览中总量指标与摘要列表统计口径不一致造成的误解：顶部数字表示所有未满足上线条件的店铺总数，下方列表只表示其中缺少商品资料的店铺。

## 范围

- 将顶部指标和“优先工作”中的“上线阻断”改为“未满足上线条件”。
- 为顶部指标提供 Tooltip，说明未满足上线条件包括缺少商品资料、缺少价格配置、缺少已审核知识、API 未完成接入等情况。
- 将下方“上线阻断摘要”改为“缺少商品资料的店铺”。
- 在列表标题附近展示说明：“以下店铺因缺少必要配置，暂时无法上线。”
- 将列表空状态调整为商品资料口径，不再使用笼统的上线阻断表述。
- 在列表底部增加“查看全部未满足上线条件的店铺”按钮，进入现有“配置完成度”页面查看完整原因。

## 实现边界

- 保持服务端 `readiness_blockers` 聚合统计、Dashboard API 和 readiness API 不变。
- Dashboard 继续展示现有 readiness 摘要数据，不在前端重新计算总量。
- 跳转复用 System Admin 现有页内导航，不新增路由或一级菜单。
- 不改动 Customer Admin、配置完成度页面内容或其他总览卡片。
- 优先在 Dashboard 页面内实现专用列表标题、说明和按钮，避免为单一场景扩大通用 `DataTable` API。

## 交互与无障碍

- Tooltip 文案可由指针悬停发现，并通过现有指标组件的可访问标题能力暴露。
- 底部按钮使用原生 `button`，触发后切换到 `readiness` 页面。
- 有数据和空状态均保留说明及完整列表入口。

## 验收

- 页面不再显示“上线阻断”或“上线阻断摘要”。
- 顶部总数显示为“未满足上线条件”，Tooltip 完整列出四类典型原因。
- 摘要标题明确为“缺少商品资料的店铺”，并出现必要配置说明。
- 点击底部按钮后显示“配置完成度”页面。
- Dashboard 组件测试覆盖新文案、Tooltip、空状态和跳转行为。
