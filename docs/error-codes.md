# 错误编号

本文只登记 ecommerce-cs-agent 当前 open_erp integration 与 Customer Admin launch exchange 链路的稳定错误编号。现有 `error.code` 保持兼容；新增 `errorId` 供 open_erp_agent 和截图排障定位。

| 错误编号 | 用户可见提示 | 触发条件 | 排查步骤 | 对应代码文件 | 函数/位置 | 相关接口 | 上游错误映射 |
|---|---|---|---|---|---|---|---|
| ECS-OE-001 | open_erp integration token is missing or invalid | `/v1/integrations/open-erp/*` 请求缺少或传错服务间 token | 检查调用方是否使用 `OPEN_ERP_INTEGRATION_TOKEN` 对应 Bearer token；确认运行时 Secret 和部署环境一致；不要打印 token 明文 | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `require_service_auth`、open_erp integration routes | `POST /v1/integrations/open-erp/provision`、`PATCH /v1/integrations/open-erp/connectors/{connector_id}`、`POST /v1/integrations/open-erp/admin-launch-tickets` | open_erp_agent 映射为 `OEA-AICS-003` 的 `upstreamErrorId` |
| ECS-OE-002 | open_erp connector is not bound or active | admin launch ticket 请求对应 platform/store/account 没有 active connector，或 connector 不存在 | 先调用 provision；检查 connector status；确认 Agent pod 重启后 connector binding 是否恢复；核对 `platform`、`external_store_id`、`platform_account_ref` | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `issue_admin_launch_ticket`、`patch_open_erp_connector` | `POST /v1/integrations/open-erp/admin-launch-tickets` | open_erp_agent 映射为 `OEA-AICS-003` 的 `upstreamErrorId` |
| ECS-OE-003 | open_erp integration request validation failed | provision、connector patch 或 admin launch ticket 请求缺少必填字段或字段值非法 | 对照请求体检查 `platform`、`external_store_id`、`platform_account_ref`、`status`；确认 open_erp_agent 发出的字段名没有漂移 | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `_required_text`、`patch_connector`、open_erp integration routes | `POST /v1/integrations/open-erp/provision`、`PATCH /v1/integrations/open-erp/connectors/{connector_id}`、`POST /v1/integrations/open-erp/admin-launch-tickets` | open_erp_agent 映射为 `OEA-AICS-003` 的 `upstreamErrorId` |
| ECS-LAUNCH-001 | launch token has already been used | 同一个一次性 launch token 被重复兑换 | 让客户端重新点击客户系统生成新 token；检查是否有浏览器重复提交或前端重试；确认没有共享 launch URL | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `consume_admin_launch_ticket`、`exchange_admin_launch_ticket` | `POST /v1/admin/auth/launch/exchange` | 无 |
| ECS-LAUNCH-002 | launch token has expired | launch token 超过短 TTL 后才兑换 | 重新从 open_erp_agent 客户端打开客户系统；检查本机和服务端时间；确认网络/前端没有卡住旧 token | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `consume_admin_launch_ticket`、`exchange_admin_launch_ticket` | `POST /v1/admin/auth/launch/exchange` | 无 |
| ECS-LAUNCH-003 | launch token was not found | 兑换了不存在的 launch token，或 Agent pod 重启后内存票据丢失 | 重新签发 launch ticket；检查 Agent pod 重启记录；后续若改持久化票据，核对存储连接 | `ecommerce_cs_agent/services/open_erp_integration.py`、`ecommerce_cs_agent/api/app.py` | `consume_admin_launch_ticket`、`exchange_admin_launch_ticket` | `POST /v1/admin/auth/launch/exchange` | 无 |
| ECS-LAUNCH-004 | launch_token is required | Customer Admin `/launch` 兑换请求没有提交 `launch_token` | 检查 `/launch?token=...` URL 是否被截断；检查前端 LaunchExchange 是否正确读取 query token；确认请求体字段名是 `launch_token` | `ecommerce_cs_agent/api/app.py` | `exchange_admin_launch_ticket` | `POST /v1/admin/auth/launch/exchange` | 无 |
